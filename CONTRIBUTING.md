# Contributing to Notebud

Thank you for considering contributing to Notebud! Please read these guidelines to help us keep the project consistent and high quality.

## What Can You Contribute?
- Code (features, bug fixes, improvements)
- Documentation
- Bug reports

## Getting Started
- Base your work on the `main` branch.
- For any change, **open an issue first** to discuss your idea or report a bug.
- Fork the repository and create a new branch for your contribution.

## Code Style
- Use the project’s code style: run `composer lint` before submitting code.
- Code should be auto-formatted using Pint (run `composer lint` to fix style automatically).
- Keep your code clean and readable.

## Commit Messages & Pull Requests
- Commit messages **must be descriptive** and use only lowercase letters (no capitalization).
- Pull request titles should also be descriptive and all lowercase.
- Group related changes in a single commit when possible.

## Tests
- Both unit and feature tests are required for code contributions.
- To run tests locally, use:
  ```bash
  php artisan test
  ```
- Make sure all tests pass before submitting your pull request.

## Review & Merging
- All pull requests will be reviewed before merging.
- Please be patient—maintainers will review as soon as possible.

## Code of Conduct
- Be respectful and constructive in all communications.
- No harassment, hate speech, or discrimination will be tolerated.

## License
By contributing, you agree that your contributions will be licensed under the MIT License.
